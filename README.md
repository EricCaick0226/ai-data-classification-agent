# AI 数据分类分级项目

`ai-data-classification-agent` 是一个面向实习准备的入门项目。  
目标是把一个 CSV 文件转成一份可审阅的数据分类分级报告。

## 这个项目要解决什么问题

给定一个 CSV，系统需要回答：

- 每个字段是什么类型的数据
- 字段属于哪一类数据（如个人信息、敏感信息等）
- 风险等级是什么（低/中/高）
- 为什么这样判断
- 是否建议人工复核

## 输入与输出

- 输入：一个 CSV 文件
- 输出：一份 Markdown 报告，包含每个字段的分类、等级、理由和建议

## 核心处理流程

1. 读取 CSV 文件
2. 提取字段信息（字段名、数据类型、缺失值、样例值）
3. 根据规则进行字段类型判断
4. 完成分类分级并给出理由与建议
5. 生成 Markdown 报告

## 项目阶段

### 阶段 1：规则版分类器（当前重点）

先使用 Python + pandas 做出一个可运行闭环。

示例规则：

- `email` -> 个人信息（PII）-> 中风险
- `phone_number` -> 个人信息（PII）-> 中风险
- `password` -> 敏感信息（Sensitive）-> 高风险
- `id_card` -> 身份证件信息（Confidential）-> 高风险

### 阶段 2：增强模糊字段判断

在规则版分类器完成后，再处理仅靠字段名难以判断的字段。

例如：

- `contact`
- `user_info`
- `credential`
- `identifier`

这些字段可能无法只靠字段名判断，需要结合样例值和上下文分析。

### 阶段 3：工具化封装

将 CSV 分析、规则分类、报告生成拆成可调用工具，便于组合使用。

未来这些工具可以包括：

- CSV 分析工具
- 字段分类工具
- 风险分级工具
- 报告生成工具

### 阶段 4：最小自动化流程

在前面步骤都稳定后，再把这些能力串成最小自动化流程。

流程不追求复杂，只需要能根据任务判断：

- 是否需要读取 CSV
- 是否需要执行字段分析
- 是否需要进行分类分级
- 是否需要生成报告
- 哪些字段需要人工复核

## 当前里程碑
